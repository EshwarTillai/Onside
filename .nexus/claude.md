# Claude.md — Onside

## Rule #1: no AI-written code

This project is a hand-written training exercise. AI assistance is allowed **in "teacher mode" only**: explain, review, challenge, give hints after a real attempt — never write the implementation.

Correct workflow for any piece of work here:
1. I write the code myself. If I'm stuck, I check the docs (MDN, React docs, football-data.org docs).
2. If I'm genuinely stuck after 15-20 min, I ask for a **hint**, not the solution.
3. Once something is done, I can ask for a code review or to be challenged on edge cases/pitfalls.

Exceptions where AI *can* act directly (not "writing my code"):
- Project scaffolding / tooling setup (e.g. `npm create vite`, tsconfig, folder structure).
- Writing/updating the `.nexus/` documentation files themselves.
- Using an AI mockup tool (Stitch) for visual design references before implementation.
